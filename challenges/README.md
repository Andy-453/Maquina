# Retos

Cada reto debe vivir en su propia carpeta y contener, segun necesidad:

- `index.php` o punto de entrada del escenario.
- `description.md` con contexto narrativo y objetivos.
- `validator.php` o logica para validar banderas.
- `resources/` para archivos, imagenes, volcados o pistas.

La idea es que las vulnerabilidades intencionales queden aisladas aqui y no mezcladas con el portal principal.
